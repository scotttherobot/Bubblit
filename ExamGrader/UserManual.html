User Manual
Adding an Exam File for Grading
  1. Preparation
    • Ensure you have a local copy of the exam you want to add. The exam must be in Portable Document Format (PDF) with the first exam representing the answer key and every subsequent exam representing the forms to be graded.
    • Make sure the local copy of the exam is properly named, the resulting solution file will also be named after this file.
    • Make sure the local copy of the exam is in the directory you want the resulting score files to be outputted. The resulting score files will be outputted to the same directory as the exam file.
  2. Adding the Exam File
    • Press the "Add Exam File" button available in the graphical user interface.

    • Navigate to the location of your exam file using the file explorer interface. The result should look like the following:
    • Select your exam file by either double clicking it, or selecting it and pressing 'Open'.
Removing an Exam File from the Grading Queue
  1. Preparation
    • Make sure you already have an added exam that you want removed.
    • Understand that removing an exam from the queue will mean that it will not be graded.
    • Understand that this step cannot be undone but the exam file can be added again.
  2. Removing the Exam File
    • Select the exam file you want removed from the exam file queue. The file will then be highlighted and the "Remove Exam File" button will be available for clicking.

Selecting/Modifying a Roster File
  1. Preparation
    • Understand that the purpose of the roster file is to identify the students taking the exam and provide this student information in the resulting score files.
    • Ensure that you have a local roster file containing the student information of the individuals who have taken the exams in your exam files. The roster file must be in the Tab Separated Values (TSV) format.
    • This roster file can be acquired through the California Polytechnic State University of San Luis Obispo's website with sufficient permissions.

Command Line Operation
  1. Preparation
    • Make sure you are on a Linux or Mac platform.
    • Ensure that you have Java Runtime Environment version 1.7 or above installed.
    • Make sure you have access to a local copy of the Bubblit jar file which can be found here.
  2. Opening the Command Line
    • On a Mac platform, open the Applications folder followed by the Utilities folder. You can find the Terminal in this directory. You may also find the Terminal through the Spotlight search.
    • On most Linux platforms, the terminal can be directly opened through the desktop's default application menu or its corresponding search system.
  3. Running the Software
    • Make sure your current directory in the command line contains the Bubblit jar file.
    • Run the command "java -jar Bubblitv2.0.jar".
    • You can include any of the following flags in the command to modify the program's behavior:
      • -v
        Display version and team info.
      • -h
        Display this syntax help info
      • -i
        Path to file of scanned exams in PDF format. Accepts relative or absolute paths (e.g., /tmp/myscans.pdf).
      • -r
        Path to student roster TSV file (relative or absolute path). This is required
      • -o
        Path to folder in which result files will be placed (relative or absolute path).
